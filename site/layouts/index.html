What Does "Trustless" Mean?

In traditional voting, you must trust election officials to count votes honestly and protect your privacy. TVS replaces that trust with mathematical proof.

Traditional Voting

  • ✗ Trust administrators won't peek at votes
  • ✗ Trust database operators won't modify records
  • ✗ Trust software has no backdoors
  • ✗ Trust a single key holder won't be compromised
  • ✗ No way to independently verify your vote

TVS Trustless Voting

  • ✓ Server cannot see votes (encrypted in browser)
  • ✓ Tampering is cryptographically detectable
  • ✓ All code is open source and auditable
  • ✓ Key split among 5 trustees (need 3 to decrypt)
  • ✓ Verify your vote with Merkle proof
{{/* The Problem */}}

The Problems We Solve

Vote Privacy

If administrators can see how you voted, they can be compelled to reveal it. TVS encrypts your vote in the browser—the server only sees ciphertext.

Single Point of Failure

One key holder = one target. TVS splits the election key among 5 trustees. Need 3 to decrypt. No single person can compromise the election.

Vote Integrity

How do you know votes weren't modified? TVS stores votes in a Merkle tree. Any change is cryptographically detectable.
